SQL

KharmonMonitor can test your SQL servers. It supports Interbase servers.
Servers are checked by logging into (and out) of the server. You should specify the server name, database, user name and password.
For Interbase you can specify the protocol (TCP, SPX, or NetBEUI).

In order to monitor Interbase servers KharmonMonitor uses the Interbase Call Interface (gds32.dll), you should have the Interbase client software installed on the computer.

Interbase/Firebird test methods may display "unknown" status and appropriate error message in the Reply field when it was impossible to perform the check (e.g. when client library is not installed).
