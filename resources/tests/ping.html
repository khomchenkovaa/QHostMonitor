Ping

The ping command verifies connections to remote computers, routers and other network components by sending ICMP (Internet Control Message Protocol) echo packets to the remote component and listening for echo reply packets. It's best to use the Ping method to check a remote connection in general. To check specific services and conditions, KharMonitor provides a number of special test types.
The Ping test has the following options:

Address
Here you should provide the domain name (e. g. www.yahoo.com) or IP address (e. g. 204.71.200.68) of the host that you wish to monitor. Also you may provide IPv6 addresses (e.g. fe80::370:ff56:fed5:22) and specify hostname with suffixes ::ipv4 or ::ipv6 (e.g. www.google.com::ipv4 or www.6bone.net::ipv6).
If you specify hostname without any suffix, KharMonitor will try to resolve name into IP address using IPv4 protocol. If name cannot be resolved by IPv4 protocol, KharMonitor will try to use IPv6 protocol. If you specify hostname with ::ipv4 (or ::ipv6) suffix, KharMonitor will use IPv4 (or IPv6) protocol only.
Note: on Windows NT 4.0, Windows 2000 and Windows XP SP1 IPv6 protocol is not supported. You may use menu Tools -> Local Info to check is your system IPv6 ready.
Also, a range of IPv4 addresses (e.g. 10.10.1.100 - 10.10.1.254) rather than a single address can be specified. In this case, KharMonitor will create a separate test for each of the addresses within that range.

Timeout
Specify a timeout interval in milliseconds. If you set parameter to 0, KharMonitor will use value of the global Timeout parameter specified on Ping/Trace page in the Options dialog.

Packet size
Specify the size of echo packets. This parameter is optional. If it's value is 0 (default value), KharMonitor will use global Packet Size parameter specified on Ping/Trace page in the Options dialog.

Packets
Specify the number of echo packets. If you set parameter to 0, KharMonitor will use value of the global "Packet to send" parameter specified on Ping/Trace page in the Options dialog.

Don't fragment
This option tells KharMonitor to set "Do Not Fragment" ICMP bit.

Time to Live
Time to Live parameter is common for all Ping tests, specify this parameters on Ping/Trace page in the Options dialog.

Alert if loss ratio N% or more
A packet loss threshold can be specified for a Ping test, based on which the test status is determined.

Alert if Jitter over N ms
Jitter - amount of variation in response time, calculated by using the following formula Jitter = SquareRoot( SumOf((ReplyTime[i] - AverageReply)^2) / (ReceivedPacketCount-1) )
(when this option enabled, KharMonitor will send at least 3 ICMP packets)

Display mode
Tells KharMonitor what kind of data should be displayed in the Reply field:
 Reply time - the "Reply" field will represent average reply time
 Lost % - display percentage of lost packets
 Received % - display percentage of received packets
 Jitter - amount of variation in response time
Note: Test Info window may show Loss ratio and Jitter in addition to selected "reply" value

KharMonitor may set 6 statuses for Ping tests:
"Ok" this status used when Jitter" option is enabled and Jitter time within specified limit
(when "Alert if lost ratio" option enabled, it should be within specified limit as well)
"Host is alive" this status used when "Jitter" option was not enabled and loss ratio within specified limit
"Bad" this status means either Jitter or Loss ratio is out of the limits
"No answer" KharMonitor sets "no answer" status when does not receive any response from target system
"Unknown host" host name could not be resolved to IP address
"Unknown" test performed by Remote Monitoring Agent and there is some RMA related error (e.g. KharMonitor cannot connect to the agent)
